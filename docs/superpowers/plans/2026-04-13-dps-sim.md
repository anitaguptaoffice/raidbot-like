# 恶魔术 DPS 模拟 Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 构建一个仿照 Raidbots 的首版在线 DPS 模拟，只支持恶魔术、SimC profile 粘贴输入、`Patchwerk`/`Dungeon Slice`/`Target Dummy` 三种 Fight Style，以及 `Num Enemies` 参数。

**Architecture:** 新建一个最小但可扩展的 Web 应用，前端负责输入与结果展示，后端负责任务创建与查询，执行层调用 `SimulationCraft` 完成异步模拟。任务模型从首版开始保留 `jobType` 字段，为后续 `Top Gear` 扩展预留边界。

**Tech Stack:** 推荐 Next.js 或 Vite + React 前端、Node API、SQLite/Postgres 任务表、`SimulationCraft` CLI。

## 验收标准

- 用户可以提交一份恶魔术 `SimC profile`
- 用户可以选择 `Patchwerk`、`Dungeon Slice`、`Target Dummy`
- 用户可以单独设置 `Num Enemies`
- 前端能正确展示任务状态与结果
- 后端能保存原始输出并在失败时返回错误
- 数据模型已为 `Top Gear` 预留 `jobType` 扩展位

## 假设与默认值

- 首版运行环境允许安装并调用 `SimulationCraft` CLI
- 首版采用异步轮询，不做 WebSocket
- 首版不做用户系统和历史任务列表
- 首版不做 Battle.net 导入
- 首版只支持恶魔术，其他专精输入直接报错
